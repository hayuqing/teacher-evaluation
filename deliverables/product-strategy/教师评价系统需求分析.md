## 📌 TL;DR（执行摘要）

- **核心目标**：为九年一贯制学校建设**认知负荷匹配**的教师评价问卷系统——学生端（匿名告知→选年级[学段选项卡切换]→选班级→逐位评价保存→总确认提交）+ 管理端（角色自适应数据看板，三类用户三个首页）
- **关键决策**：学生端移除跳过/分批/稍后评价，改为强制逐位逐维度完成；单位教师操作为“完成/保存”，最终确认页才提交整份问卷；学段选择改为年级页选项卡（入口页仅匿名告知且隐藏开发预览控件）；管理端独立；教师切换增强动画；管理端角色自适应首页路由（登录即分流）；数据置信度三层标注（低段定性参考/中段半定量/高段发展性评价）；教师保护机制（等级标签替代分数、群体参照替代排名）
- **差异化壁垒**：学生端——年级-班级-教师自动映射引擎 + K-9三档适龄维度/交互 + 等级选项制 + 反向校验题 + 强制完成机制；管理端——角色自适应首页 + 分学段递进维度看板 + 置信度分层视觉体系 + 发展性反馈+教师保护组合；两端壁垒组合在市场上无竞品覆盖
- **端间依赖**：管理端完全依赖学生端产生的评价数据（M3核心看板须在M1学生端MVP完成后才能开发）
- **下一步**：本周须锁定Q-3（4-6年级选项级数）、Q-10（后端等距vs不等距映射）、Q-A1（教师群体参照解锁机制）和Q-A2（催促通知渠道）

## 第1章 产品目标

### 问题陈述

九年一贯制学校缺乏科学的、符合各学段学生认知发展水平的教师评价工具。现有方案采用统一数值量表，忽视了低龄学生对抽象数字的理解局限，导致数据信度低下、光环效应严重。亟需一套**认知负荷匹配**的评价体系，让每个学段的学生都能以自己能理解的方式真实表达感受。

同时，教师评价系统的管理端仅有基础仪表盘原型（4个KPI卡片+1个雷达图+任务列表）。三类管理端用户（教务管理员、校领导、被评价教师）面临不同痛点：管理员过程失控焦虑、领导缺乏参照系的孤值困扰、教师的排名恐惧和信息羞辱感。亟需一套**角色自适应、数据置信度分层、教师发展导向**的管理端数据看板。

### 目标用户

**学生端用户**：
- **主要**：1-9年级在校学生（评价主体）
- **次要**：班主任（组织实施）

**管理端用户**：

| 角色 | 核心需求 | 使用频率 | 首页定位 |
|------|---------|---------|---------|
| 教务管理员 | 过程掌控：创建任务→监控进度→清洗数据→发布结果 | 评价期间每日 | 任务管理中心 |
| 校领导/决策层 | 决策支持：全校概况→识别薄弱环节→改善优先级 | 每学期1-2次 | 全校概况仪表盘 |
| 被评价教师 | 发展反馈：个人雷达→维度解读→改善建议 | 结果发布后1-2次 | 我的评价 |

### 成功指标

**学生端指标**：

| 编号 | 指标 | 目标值 | 依据 |
|------|------|--------|------|
| G1 | 问卷完成率 | ≥85%（全学段） | 整体覆盖 |
| G2 | 低段(1-3)独立完成率 | ≥60%（等级选项+emoji辅助下，不含数值理解门槛） | 低段可达性 |
| G3 | 中高段(4-9)独立完成率 | ≥80% | 高段可达性 |
| G4 | 反向校验题异常检出率 | ≤15%（低段随意作答识别） | 数据质量 |
| G5 | 课堂管理维度信度 | ICC2 ≥ .60（参照Fauth等2014基准） | 维度信度 |
| G6 | 教师发展性反馈采纳率 | ≥40%（第一学年） | 价值转化 |

**管理端指标**：

| 编号 | 指标 | 目标值 | 依据 |
|------|------|--------|------|
| A1 | 教师查看率 | ≥50% | 结果发布后7天内教师登录查看比例 |
| A2 | 改善建议采纳率 | ≥40% | 教师标记"已采纳"的建议比例 |
| A3 | 管理员任务创建效率 | ≤10分钟/任务 | 从创建到发布全流程耗时 |
| A4 | 参与率实时追踪延迟 | ≤30秒 | WebSocket推送延迟 |
| A5 | 异常检出自动化率 | ≥80% | 系统自动标记vs人工标记的比例 |
| A6 | 报告导出完成时间 | ≤30秒/PDF | 异步生成+邮件通知 |

> 十二目标正交：G1-G6关注学生端体验与数据质量，A1-A6关注管理端效率与价值转化。

## 第2章 用户故事

**学生端用户故事**：

- **US-1**：作为1年级学生，我想看到😊/😐/😟和"很好/还行/不好"这样的文字来评价老师，因为我还不理解分数是什么 → 确保低段量表为3级文字+emoji选项，**不出现任何数字**
- **US-2**：作为5年级学生，我想用"很好/比较好/一般/不好"来评价，因为我能区分程度但不需要用精确数字 → 确保中段量表为3-4级文字标签选项
- **US-3**：作为8年级学生，我想用"很好/较好/一般/较差/很差"来细致评价并写建议，因为我已有成熟判断力 → 确保高段量表为5级李克特选项+可选开放题
- **US-4**：作为班主任，我想一键分发适合本班学段的问卷，不用手动配置 → 系统根据年级自动匹配维度与量表
- **US-5**：作为教务管理者，我想看到各维度均分和趋势，但知道低段数据是定性参考而非精确分数 → 报告标注数据置信度层级
- **US-6**：作为教师，我想收到发展性反馈帮助改进教学，而不是被一张分数表排名 → 评价结果仅供发展性使用

**管理端用户故事**：

- **US-A1**：作为教务管理员，我想一键创建评价任务并看到全校各班的实时参与率，这样我不用逐班催收和手工统计 → 管理员首页=任务管理中心，参与率实时WebSocket推送
- **US-A2**：作为教务管理员，我想系统自动标记异常问卷（全选同一等级、反向校验矛盾、超短提交时间），这样我不需要逐条人工筛查 → 异常筛查模块+自动化规则引擎
- **US-A3**：作为校领导，我想一页看到"全校哪个维度最弱、哪个年级段需要重点关注"，数据自动叠加校级均值和上学期参照线，这样我不需要自己判断"84分算好还是差" → 全校概况页+改善优先级自动排序+自动参照系
- **US-A4**：作为校领导，我想看到所有教师的维度热力图一眼识别薄弱区域，但不能看到教师排名——排名会扭曲评价动机 → 热力图用"高于/接近/低于均值"色彩编码，不做绝对排名
- **US-T1**：作为教师，我想看到自己的雷达图上叠加"年级平均"参照线，但不看到任何排名或裸分数，这样我知道自己在群体中的位置但不被羞辱 → 群体参照线默认关闭，主动解锁时叠加匿名均值线
- **US-T2**：作为教师，我想看到"课堂管理=表现优秀（高于年级均值0.5分）"这样的等级标签而非"3.8分"，这样我理解结果不需要翻译 → 维度用等级标签而非数字
- **US-T3**：作为教师，我想收到具体的改善建议（"课堂管理维度有提升空间，建议尝试小组讨论增加学生参与感"）而非笼统的"你的分数较低" → 发展建议模块+建议库规则映射
- **US-T4**：作为教师，我想看到自己最近3个学期在"课堂管理"维度上的进步曲线，这样我知道改善措施是否有效 → 成长轨迹模块+纵向折线图

## 第3章 用户研究洞察（来源：瑞思 + 调研报告）

### 认知负荷匹配原则（学生端核心框架）

评价量表的设计必须匹配评价主体的认知发展阶段。皮亚杰认知发展理论指出：

| 学段 | 认知阶段 | 评价能力特征 | 量表设计 implications |
|------|---------|-------------|----------------------|
| 1-3年级 | 具体运算早期(7-9岁) | 对"等距数字"理解不稳定，区分"7分还是8分"本质是抽象判断，产生大量随机误差；倾向整体好感（光环效应） | 3级文字选项+emoji，不做数字区分 |
| 4-6年级 | 具体运算成熟期(10-12岁) | 能做程度区分但尚未稳定量化，可理解"比较好"vs"一般"的语义差异 | 3-4级文字标签选项，可选星级辅助 |
| 7-9年级 | 形式运算期(13-15岁) | 具备抽象判断力，可做多维度细致评价和反思性表达 | 4-5级李克特选项+可选开放题 |

### 核心洞察（统一编号 I-1 ~ I-14）

**学生端洞察（I-1 ~ I-9）**：

| # | 洞察 | 产品影响 |
|---|------|---------|
| I-1 | **认知发展基础**：6-9岁对等距数字理解不稳定，数字打分本质是超出认知发展水平的抽象判断 | 量表改为等级选项制，前端不显示数字 |
| I-2 | **语言能力**：低段学生阅读理解有限，问卷表述必须儿童化、具象化 | 维度表述用"老师讲得好不好"而非"教学能力" |
| I-3 | **注意力容量**：低段注意力约15分钟，单次问卷≤20题 | 进度反馈+控制题量 |
| I-4 | **技术匿名≠心理匿名**：匿名设计须贯穿UI每一个细节 | 入口匿名告知+token机制+UI无身份暗示 |
| I-5 | **评价疲劳是数据质量最大威胁** | 逐位滑入式呈现+强进度反馈 |
| I-6 | **光环效应**：低段学生倾向在所有维度都选"很好"，维度间高度相关 | 低段设反向校验题；数据标注为定性参考 |
| I-7 | **等级选项制优于数值打分制**：学术共识，即便是成人成熟量表也用文字标签Likert而非裸数字 | 前端只显示文字标签+emoji，后端等距映射 |
| I-8 | **课堂管理是最稳定可测维度**：ICC2=.63-.82 | 低段优先保留课堂管理维度 |
| I-9 | **师德师风超出学生评价能力**：应由同行/家长/管理层多主体评定 | 学生评教不含师德维度 |

**管理端洞察（I-10 ~ I-14）**：

| # | 洞察 | 管理端设计影响 |
|---|------|---------------|
| I-10 | **三类用户"数据翻译"需求截然不同**：管理员→操作翻译（参与率→催促动作）、领导→决策翻译（分数→"高于/低于平均"）、教师→成长翻译（等级→"哪里可以更好"） | 角色自适应首页架构，三角色共享后端API但前端呈现逻辑完全分层 |

| I-14 | **分学段量表决定了看板必须分学段聚合**：1-3年级4维/定性参考、4-6年级5维/半定量、7-9年级6维/发展性评价——三个学段数据性质完全不同 | 全校概况分学段呈现（三个独立雷达图+学段切换器）；跨学段对比标注"仅可比共同维度"；低段标注"定性参考" |

### 5A. 学生端数据依据与评价维度体系

#### 数据依据

| 编号 | 依据 | 来源 | 产品决策 |

| D-2 | 课堂管理是小学生最稳定可测维度（ICC2=.63-.82） | Fauth et al., 2014 | 低段优先保留课堂管理 |
| D-3 | **分学段递进式等级选项(3→4→5级)，后端等距映射为分值** | 皮亚杰理论；SAT/课程评教均用文字标签Likert | 替代原"统一5分制Likert"方案 |

| D-6 | 教学质量三维度框架：课堂管理、支持性氛围、认知激活 | Praetorius et al., 2016; Fauth et al., 2014 | 维度体系学术基础 |
| D-7 | 师德师风超出学生评价能力，应由多主体评定 | "四有"好老师标准；Klieme框架 | 学生评教排除师德 |

#### 评价维度体系（核心壁垒，完整保留）

**设计原则：低段做减法（4维、3级选项）、中段做过渡（5维、3-4级）、高段做加法（6维、4-5级）**

> **学术来源**：维度框架基于Praetorius等(2016)教学质量三维度（课堂管理、支持性氛围、认知激活）和Fauth等(2014)小学生评教信度数据。低段不含认知激活维度，因其对低龄学生最难准确判断；中段引入简化版"启发思考"；高段完整纳入"启发思维"对应认知激活。

##### 低学龄段(1-3年级) — 4个维度

| 维度 | 儿童化表述 | 选项 | 学术映射 |
|------|-----------|------|---------|
| 老师讲得好不好 | "老师讲课，你能听明白吗？" | 很明白😊 / 有点明白😐 / 听不懂😟 | →教学能力(具象化) |
| 课堂开不开心 | "上这位老师的课，你觉得开心吗？" | 很开心😊 / 还行😐 / 不开心😟 | →课堂管理(情感锚定) |
| 老师对你好不好 | "老师对你好不好、有没有耐心？" | 很好😊 / 一般😐 / 不好😟 | →师生关系(支持性氛围) |
| 课堂乖不乖 | "我们上课时，大家乖不乖、听不听话？" | 很乖😊 / 还行😐 / 乱糟糟😟 | →教学态度与规范(秩序锚定) |

后端映射：很好/很明白/很开心/很乖 = 3分；还行/有点明白/一般 = 2分；不好/听不懂/不开心/乱糟糟 = 1分（等距3级映射）

##### 中学龄段(4-6年级) — 5个维度

| 维度 | 表述 | 选项 |
|------|------|------|
| 教学讲解 | "老师讲的内容，你容易听懂吗？" | 很好/比较好/一般/不好 |
| 课堂管理 | "老师上课时，班里秩序好不好？" | 很好/比较好/一般/不好 |
| 作业与反馈 | "老师布置的作业合理吗？批改认真吗？讲评及时吗？" | 很好/比较好/一般/不好 |
| 关心与鼓励 | "老师关心你吗？你提问时老师耐心解答吗？" | 很好/比较好/一般/不好 |
| 启发思考 | "老师会鼓励你们多思考、多动手吗？" | 很好/比较好/一般/不好 |

后端映射：很好=4 / 比较好=3 / 一般=2 / 不好=1（等距4级映射）；可选配少量3-4星评级辅助

##### 高学龄段(7-9年级) — 6个维度

| 维度 | 表述 | 选项 |
|------|------|------|
| 教学能力 | "老师讲解清晰、重点突出，能帮你理解难点吗？" | 很好/较好/一般/较差/很差 |
| 课堂管理 | "课堂纪律有序，老师能有效组织教学活动吗？" | 很好/较好/一般/较差/很差 |
| 启发思维 | "老师是否善于启发思维、鼓励提问与探究？" | 很好/较好/一般/较差/很差 |
| 师生关系 | "老师尊重、公平对待每位同学，耐心解答问题吗？" | 很好/较好/一般/较差/很差 |
| 作业反馈 | "作业布置合理，批改认真、讲评及时吗？" | 很好/较好/一般/较差/很差 |
| 教学态度 | "老师是否认真负责、不迟到早退、备课充分？" | 很好/较好/一般/较差/很差 |

后端映射：很好=5 / 较好=4 / 一般=3 / 较差=2 / 很差=1（等距5级映射）；可加1道开放性建议题

#### 题量方案

| 学段 | 每教师题量 | 评价教师数 | 总题量 | 策略 |
|------|-----------|-----------|--------|------|
| 1-3年级 | 4维度×1题 = 4题 + 1道反向校验题 = 5题 | 5位 | 20+1校验 = 21题 | 单次完成（4维×5=20题在阈值内；反向校验题不计入负荷） |
| 1-3年级(含总体题) | 4+1总体+1校验 = 6题 | 5位 | 25+1 = 26题 | 低段不含总体题（4维度已覆盖核心感知） |
| 4-6年级 | 5维度×1题 = 5题 + 1总体 = 6题 | 4位 | 24题 | 控制题量 |
| 4-6年级(不含总体) | 5题 | 4位 | 20题 | 单次完成 ✓ |
| 7-9年级 | 6维度×1题 + 1总体 + 1开放 = 8题 | 3位 | 24题 | 控制题量 |
| 7-9年级(不含开放) | 6+1 = 7题 | 3位 | 21题 | 单次完成 ✓ |

> **设计决策**：低段不含总体满意度题（4维度已覆盖核心感知，总体题对低段意义有限且增加负荷）；中段总体题可选；高段建议包含总体题+开放题。

## 第8章 Non-goals（明确不做什么）

| # | 不做什么 | 原因 | 来源 |
|---|---------|------|------|
| NG-1 | 不做通用问卷编辑器（自由添加题目） | 核心价值在标准化维度体系+自动映射，自由编辑破坏数据可比性和适龄设计 | 学生端 |
| NG-2 | 不做跨校/学区级评价平台 | v1聚焦单校场景 | 学生端 |
| NG-3 | 不做教师排名和奖惩依据 | 评教数据信度有限，排名扭曲评价动机 | 两端共有 |
| NG-4 | 不做跨学段横向对比 | 各学段维度和量表不同，不具备可比性 | 两端共有 |
| NG-5 | 不做实时/视频评价 | 与场景无关，增加复杂度 | 学生端 |
| NG-6 | 不做家长评价通道(v1) | 家长维度体系需独立设计 | 学生端 |
| NG-7 | 不做AI自动分析报告(v1) | 依赖数据积累 | 学生端 |
| NG-8 | **不做纯数值打分面向学生** | 学术共识：文字标签Likert优于裸数字，低龄对数字理解不稳定 | 学生端 |
| NG-9 | **学生评教不含师德师风** | 师德超出学生评价能力，由多主体评定 | 两端共有 |
| NG-10 | 不做移动端App(v1) | 桌面端优先Web适配（评价场景为学校机房电脑端，原型断点1024px以下为平板/小屏兼容降级） | 两端共有 |
| NG-11 | 不做裸分数展示面向教师 | 教师保护机制核心：等级标签替代分数 | 管理端 |
| NG-12 | 不做管理员查看教师个体排名 | 管理员看热力图相对位置而非绝对排名 | 管理端 |
| NG-13 | 不做自动奖惩决策 | 评价数据仅供发展性使用，不做自动决策避免伦理风险 | 管理端 |
| NG-14 | 不做跨学段混合聚合 | 各学段维度和量表不同，不具备可比性 | 管理端（与NG-4合并，独立保留以强调管理端视角） |

> **去重说明**：NG-3（排名/奖惩）、NG-4（跨学段对比）、NG-9（师德排除）、NG-10（移动端App）为两端共有Non-goal，统一编号后去重保留。NG-4与NG-14实质相同但视角不同（学生端=不让用户选跨学段对比，管理端=不让管理员做全校混合聚合），合并为NG-4但强调两端视角。
